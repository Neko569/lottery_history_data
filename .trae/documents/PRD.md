# 彩票开奖历史查询及随机生成网页 - 产品需求文档 (PRD)

## 1. 产品概述

一个可部署于 GitHub Pages 的纯前端彩票开奖历史查询与随机号码生成网页，默认展示体彩大乐透开奖记录，支持双色球，提供走势图与分屏对比查看能力。
- 解决彩票爱好者查看历史开奖、分析号码走势、随机生成投注号码的需求
- 目标用户为彩票研究爱好者，价值在于开源数据可视化与便捷的移动/桌面双端体验

## 2. 核心功能

### 2.2 功能模块
1. **主页面（单视图）**：顶部控制栏、开奖历史列表、分页器、随机生成器、走势图（电脑端）
2. **主页面（分屏视图）**：左右双栏分别展示大乐透与双色球

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|-----------|-------------|---------------------|
| 主页面 | 顶部控制栏 | 彩种切换、分屏开关、每页条数下拉(10/25/50/100)、数据源刷新、手动上传入口 |
| 主页面 | 开奖历史列表 | 期号、开奖日期、前区/红球号码球、后区/蓝球号码球，按期号倒序，默认25条/页 |
| 主页面 | 分页器 | 上一页/下一页、页码跳转，根据每页条数动态计算总页数 |
| 主页面 | 走势图（电脑端） | 选中彩种的前区/后区号码走势折线图，可切换显示期数 |
| 主页面 | 随机生成器 | 一键生成符合规则的号码组合（大乐透5+2，双色球6+1），支持单注/多注 |
| 主页面 | 分屏视图 | 开关开启时左右分栏，各自独立彩种、列表、走势、随机生成 |
| 主页面 | 数据源管理 | 默认拉取 GitHub raw JSON；失败时提示并支持手动上传本地 JSON 文件 |

## 3. 核心流程

用户打开页面 → 自动识别设备（PC/移动） → 拉取默认大乐透 JSON 数据 → 渲染最近25条开奖记录 → 用户可切换彩种/开启分屏/调整每页条数/查看走势图/生成随机号码。若远程数据拉取失败，提示用户手动上传 JSON 文件。

```mermaid

    "D -- 是 -->" "E[渲染开奖历史列表]"
    "D -- 否 -->" "F[提示并支持手动上传]"
    "F" --> "E"
    "E" --> "G[用户操作: 切换彩种/分屏/分页/走势/随机生成]"
```

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：深墨色背景（#0E0E12）+ 朱砂红（大乐透主色 #E63946）+ 鎏金（#D4AF37）作为强调色，营造中式彩运的精致质感
- **次级色**：靛蓝（双色球主色 #3A86FF）用于双色球区分
- **按钮风格**：圆角胶囊按钮，主操作朱砂红实心，次操作描边
- **字体**：标题用衬线显示字体（Noto Serif SC）体现庄重感，数据用等宽字体（JetBrains Mono）保证号码对齐，正文用 Noto Sans SC
- **布局**：桌面端顶部固定控制栏 + 卡片式内容区；移动端单列堆叠
- **号码球**：圆形渐变球体，前区/红球朱砂红渐变，后区/蓝球靛蓝渐变，带微妙高光

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI 元素 |
|-----------|-------------|-------------|
| 主页面 | 顶部控制栏 | 深色磨砂背景、彩种分段控件、分屏开关、每页条数下拉、刷新/上传图标按钮 |
| 主页面 | 开奖历史列表 | 行卡片式，期号等宽字体、号码球横排、悬停高亮、斑马纹 |
| 主页面 | 走势图 | 折线图卡片，多色折线，图例可切换，仅桌面端显示 |
| 主页面 | 随机生成器 | 卡片含生成按钮、号码球展示、单注/5注/10注切换 |
| 主页面 | 分屏视图 | 左右等宽分栏，中间细分隔线，各自独立控制栏 |

### 4.3 响应式
- 桌面优先设计，断点 1024px 以上为桌面布局（含走势图、分屏）
- 768px 以下为移动布局：隐藏走势图、单列堆叠、号码球缩小、控制栏折叠为横向滚动
- 通过 `matchMedia` / `window.innerWidth` 进行设备识别，PC 用大分辨率布局，移动端自适应
